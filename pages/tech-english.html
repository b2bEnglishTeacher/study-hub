02 — THEORY

Tech Talk Made Simple

You don't need to be a developer to understand tech language. In modern workplaces, everyone collaborates with tech teams — knowing the basics helps you communicate, make decisions, and stay informed.

💻

Frontend & Backend

Frontend is what users see (website, app interface). Backend is the server-side logic and databases. "Full-stack" means both. API (Application Programming Interface) connects systems.

"The frontend team is redesigning the checkout page, while the backend team is optimising the payment API to reduce processing time."
🔧

Development Process

Software is built in cycles: planning, development, testing, deployment. "Sprint" = a fixed development period (usually 2 weeks). "MVP" = Minimum Viable Product. "Iteration" = improving through repeated cycles.

"We're targeting an MVP launch by March, with bi-weekly sprints. The first iteration will include core features only."
🐛

Bugs, Testing & QA

A "bug" is a software defect. "QA" (Quality Assurance) tests for bugs. "Staging" = a test environment. "Production" = the live environment. "Regression" = a new bug caused by a fix.

"QA found a critical bug in staging — the login form crashes on mobile. We need to fix it before pushing to production."
🚀

Deployment & Infrastructure

"Deploy" = releasing code to production. "Downtime" = when a system is unavailable. "Cloud" = remote servers (AWS, Azure). "Scalability" = ability to handle growth. "Latency" = delay in response time.

"We deployed the update at 2am to minimise downtime. The new cloud infrastructure improves scalability by 40%."

Communicating with Tech Teams

Do ask developers to explain terms you don't understand — they expect it.
Do use the correct terminology once you learn it — it builds credibility.
Do focus on understanding the "what" and "why", not the "how".
Don't pretend to understand technical concepts when you don't.
Don't use vague descriptions ("the thing is broken") — be specific about what's happening.
Don't assume all tech issues are quick fixes — ask for realistic timelines.
03 — VOCABULARY

Useful Words & Phrases

API
Application Programming Interface — connects different software systems
Frontend
The user-facing part of a website or application
Backend
The server-side logic, databases, and infrastructure
Deploy
To release code or updates to a live environment
Bug
A defect or error in software that causes unexpected behaviour
Sprint
A fixed development period, usually 1–2 weeks
MVP
Minimum Viable Product — the simplest version to launch with
QA
Quality Assurance — the process of testing software for bugs
Staging
A test environment that mirrors production
Production
The live environment that real users interact with
Scalability
The ability of a system to handle increased demand
Downtime
A period when a system or service is unavailable

05 — PRACTICE

Writing Exercises

Apply your tech vocabulary in real workplace scenarios.

Bug Report

Write a bug report

You noticed the company website's contact form isn't sending messages on mobile devices. Write a clear bug report for the development team, describing what happens, what you expected, and what device/browser you used.

Feature Request

Write a feature request

Your team needs a dashboard that shows real-time sales data. Write a feature request email to the development team explaining what you need, why it's important, and any specific requirements.

Status Update

Summarise a sprint

You're the project manager. The two-week sprint just ended. Write a summary for stakeholders covering what was completed, what's in progress, any blockers, and next steps.
