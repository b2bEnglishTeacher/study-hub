02 — THEORY

The Language of Modern Software

SaaS (Software as a Service) and cloud computing have transformed how businesses operate. Understanding this vocabulary is essential for evaluating tools, managing vendors, and participating in digital transformation projects.

☁️

Cloud Computing Basics

Cloud = accessing computing resources over the internet instead of local servers. "IaaS" = Infrastructure as a Service (servers, storage). "PaaS" = Platform as a Service (development platforms). "SaaS" = Software as a Service (ready-to-use apps).

"We migrated from on-premise servers to AWS (IaaS), use Heroku (PaaS) for development, and Salesforce (SaaS) for CRM."
💳

SaaS & Subscription Models

SaaS products are accessed via browser and paid by subscription. Key terms: "monthly/annual subscription", "per-seat pricing", "freemium", "tier" (Basic/Pro/Enterprise), "churn rate".

"We're on the Pro tier at €49 per seat per month. With 25 users, our annual cost is €14,700. The vendor offers a 20% discount for annual billing."
🔗

Integrations & APIs

"Integration" = connecting two software systems. "API" = the interface that enables integration. "Webhook" = automated notification between systems. "Zapier/Make" = no-code integration tools. "Data sync" = keeping data consistent across systems.

"We integrated our CRM with the email marketing platform via API, so new leads automatically receive the welcome sequence."
🔄

Migration & Scalability

"Migration" = moving data/systems from one platform to another. "On-premise to cloud" = moving from local servers to cloud. "Scalability" = ability to grow without performance loss. "Uptime/SLA" = guaranteed availability (e.g. 99.9%).

"The cloud migration took 6 weeks. We achieved 99.95% uptime in the first quarter, exceeding our SLA target of 99.9%."

SaaS & Cloud Do's & Don'ts

Do compare total cost of ownership, not just subscription price.
Do check SLA guarantees before signing contracts.
Do plan migrations carefully — data loss is the biggest risk.
Don't adopt tools without checking integration capabilities.
Don't ignore security certifications (SOC 2, ISO 27001) when evaluating vendors.
Don't underestimate the learning curve — factor in training costs.
03 — VOCABULARY

Useful Words & Phrases

SaaS
Software as a Service — apps accessed via browser
Cloud computing
Using remote servers over the internet for computing
Subscription
Recurring payment model for software access
Integration
Connecting two or more software systems together
API
Application Programming Interface — enables system communication
Uptime
The percentage of time a service is available
SLA
Service Level Agreement — guaranteed performance terms
Migration
Moving data or systems from one platform to another
On-premise
Software hosted on local servers, not in the cloud
Freemium
Free basic version with paid premium features
Per-seat pricing
Charging based on the number of users
Scalability
Ability to grow without losing performance

05 — PRACTICE

Writing Exercises

Apply your SaaS and cloud vocabulary in the following scenarios.

Vendor Comparison

Compare two SaaS tools

Your team needs a new project management tool. Compare two SaaS options (e.g., Asana vs. Monday.com) using vocabulary from this lesson — discuss pricing tiers, per-seat costs, integrations, and uptime guarantees.

Migration Plan

Write a cloud migration proposal

Your company wants to migrate from on-premise servers to a cloud provider. Write a brief proposal covering the timeline, risks (data loss, downtime), SLA requirements, and expected benefits (scalability, cost savings).

SaaS Pitch

Pitch a SaaS product to a client

You work for a SaaS company. Write a short pitch email to a potential client explaining your product's subscription model, key integrations, uptime guarantee, and scalability advantages.
